# Implementation Report - RRA-015: Grid Icon Layout
**Date**: 2026-03-28
**Time**: 16:24
**ID**: RRA-015

## 1. Technical Approach
- **Container**: Introduce a single `ProgramManagerWindow` struct or state.
- **Grid Logic**: 
  - Icons will be laid out in a fixed grid inside the main window.
  - Padding: 40px between icons.
  - Alignment: Top-left justified.
- **State**: `DesktopManager` will now treat `icons` as items *within* a single layout rather than independent windows.

## 2. Architectural Drift Review
- **Vibe**: This is the "True" Windows 3.11 layout. Individual windows for tools *after* launch is correct, but the "Program Manager" should be have a grid.
- **DOD**: Reusing the existing `ToolIcon` struct but ignoring its `x, y` in favor of calculated grid positions.

## 3. Implementation Plan
1. **Modify `draw()`**: 
   - Draw one large "Program Manager" window.
   - Calculate `icon_x = grid_col * 80 + padding`.
   - Calculate `icon_y = grid_row * 80 + padding`.
2. **Modify `handleEvent()`**:
   - Update hit testing to use the calculated grid positions.
3. **Refactor `main()`**: Keep tool registrations, but they will now populate the grid.
